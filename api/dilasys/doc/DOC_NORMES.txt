Normes pour les pages d'affichage:
==================================
- Les pages sont affichees en utf8 (utilisation de la fonction utf8_encode)
- Inclure un contrôle en 1 ère ligne de la page d'affichage pour ne pas pouvoir afficher directement la page sans passer par le système:
"<? include('../dilasys/pre.php'); ?>"
- Inclusion systèmatique du système anti-cache (rajout du timestamp dans les liens de type GET et 
  action="actions.php?tmsp=<?=time()?>" dans la declaration des formulaires). Ce timestamp doit être rajouté à la fin du lien
  pour ne pas interférer avec l'url rewriting
- Rajout d'un champ hidden ayant pour nom "timestamp" et pour valeur "<?=time()?>" dans les formulaires impliquant un ajout en base
- Gestion des messages de résultat d'actions affichés sur la page d'affichage ("L'article a bien été rajouté ...")
- Rajout des confirmations de suppression ("Confirmez-vous la suppression...") pour les actions impliquant une suppression en base
- Mettre des titles cohérents sur les images
- Si une page doit afficher une liste avec possibilité de modification pour chaque entrée, mettre en place un système AJAX de récupération
  des données. Ne pas mettre autant de div de modifications qu'il y a d'entrées: dans certains environnements la page ne s'affichera pas...
- dans les balises de type href, si un AJAX doit être déclenché, le mettre dans le mot clef "href" et ne pas faire de "onclick":
  "<a href="javascript: rechercher_facture('<?=$id_facture?>')" onclick="App('Patienter');">facture</a>"
- dans les balises de type href, si un traitement ponctuel doit être effectué :
  "<a href="javascript: void(0)" onclick="App('Patienter'); ...">Texte</a>"
- Le nom des champs doit correspondre avec le nom des colonnes correspondantes dans la base de donnees (par exemple on doit retrouver 
  nom_adulte dans le formulaire et dans la base)
- Appliquer la fonction "nl2br" à toute information qui a été rajoutée avec un textarea et qui doit être restituée en dehors d'un textarea
- La page d'affichage par défaut porte en général le même nom que le répertoire: si le répertoire s'appelle "annuaire", la page s'appellera "annuaire.php"
- Inclure le javascript de fermeture automatique des messages: <body onload="setTimeout('disp_messages()',3000)">
- Rajout des contrôles avec "saisie.js" sur les champs devant respecter un certain format:
  Rajouter le js de contrôle dans l'en-tête:                <script type="text/javascript" src="../scripts/saisie.js"></script>

  Rajouter dans le champ concerné (onChange, onKeyUp, ...): onChange="valideField(this, 'REEL')"
 
  Types de validation disponibles:
    NOM:      ABCDEFGHIJKLMNOPQRSTUVWXYZ -
    NONVIDE:  Le champ ne peut pas rester vide
    ALPHANUM: 0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz ,/-\340\350\371\351\352\347
    ALPHAMAJ: ABCDEFGHIJKLMNOPQRSTUVWXYZ ,/-
    ALPHA:    ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz -\340\350\371\351\352\347
    EMAIL:    ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz@.-_0123456789
    TEL:      +()0123456789
    CP:       0123456789 (et 5 chiffres attendus)
    NUM:      0123456789
    REEL:     -0123456789.,
    DATE:     format JJ/MM/AAAA (Positionne automatiquement les / qd on utilise onKeyUp)
    DATE_COMPLETE: format JJ/MM/AAAA (Vérification sur un check par exemple)
    HEURE:    format HH:MM

Normes pour la base de donnees:
===============================
- Les noms des tables sont en minuscules et les mots separes par undescore ("_")
- Les noms des colonnes sont en minuscules et les mots separes par undescore ("_")
- Les noms de tables doivent etre "regroupes" par fonctionnalite
- Les colonnes correspondant a des champ de saisie dans les interfaces doivent etre des "VARCHAR(255)"
- Les colonnes correspondant a des champs de type textarea dans les interfaces doivent etre de type "TEXT"
- Toute donnee rentrant en base sera encodee en iso-8851 (utilisation de la fonction Sql_prepareTexteStockage qui appelle utf8_decode)
- Toute donnee sortant de la base sera encodee en utf8 (utilisation de la fonction Sql_prepareTexteAffichage qui appelle utf8_encode)
- Toute date rentrant en base le sera au format anglais : YYYY-MM-DD (utilisation de la fonction Lib_frToEn)
- Toute date sortant de la base le sera au format francais : JJ/MM/AAAA (utilisation de la fonction Lib_enToFr)
- Les donnees monetaires doivent etre des "DECIMAL(8,2)"

Normes de codage:
=================
- L'indentation se fera avec une tabulation correspondant a 3 espaces
- Inclusion du contrôle des F5 systèmatique sur les actions comportant des ADD (ajouts en base)
- Log et commentaires saisis au fur et à mesure dans le code
- Utilisation de "gmdate()" en lieu et place de "date()"
- Initialiser tous les tableaux inclus dans les data_out: "$data_out['tableau'] = new Array();"
- Faire un cast devant chaque tableau dans les foreach (au cas où le tableau n'aurait pas été initialisé) : foreach((array)$tab as $row) ...
- Pour les champs de type texte, utiliser un VARCHAR(255) dans la base de données
- Inclusion d'un mécanisme de contrôle d'identifiant (lors de la manipulation de données, vérifier que les données appartiennent bien à l'utilisateur connecté)
- Typer le nom des variables : "$obj_personne" pour les objets et "$tab_personne" pour les tableaux par exemple.

Normes de nomage:
=================
- Le nom des fichiers et des répertoires doit être en minuscules et la séparation entre les mots se fera avec un underscore (afin d'éviter tout problème entre les systèmes UNIX et Windows)
- Le nom des repertoires doit représenter au mieux la fonctionnalité qu'il contient
- La page "mère" (1ère page affichée par actions.php et qui dans laquelles seront affichés les différents blocs) doit avoir le même nom que le répertoire où elle se trouve
- Les autres pages doivent être préfixées avec le même nom que le répertoire où elles se trouvent
- Rajouter le terme "bloc" pour les pages de type AJAX affichant un bloc HTML complet dans la page "mère"
- Rajouter le terme "popup" pour les pages de type AJAX affichant un bloc HTML complet dans une fenêtre de type popup
- Rajouter le terme "select" pour les pages de type AJAX affichant un champ de type select
- Rajouter le terme "auto" pour les pages de type AJAX affichant une liste de type autocomplete
